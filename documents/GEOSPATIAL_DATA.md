# Geospatial Data Reference

This document describes the shapefiles stored in `shared/Q_to_R/data/` and how they are used across apps.

All shapefiles are in **WGS84 (EPSG:4326)** projection. They are generated by the extraction scripts in `shared/Q_to_R/` from the PostgreSQL/PostGIS database.

---

## District-Wide Shapefiles

These files live directly in `shared/Q_to_R/data/`:

| File | Source | Description | Used By |
|------|--------|-------------|---------|
| `facility_boundaries.shp` | `gis_sectcode` (grouped) | Polygon boundaries for each facility (E, N, Sj, Sr, Wm, Wp) | air_sites_simple, suco_history, trap_surveillance |
| `zone_boundaries.shp` | `gis_sectcode` (grouped) | P1 and P2 zone polygons | air_sites_simple, suco_history, trap_surveillance |
| `sections_boundaries.shp` | `gis_sectcode` | Individual section polygons (3207 sections) | trap_surveillance |
| `recent_trap_locations.shp` | `dbadult_insp_current` | Point locations of recent adult traps | trap_surveillance |
| `VectorIndexAreasA2025.shp` | Custom grouping | 12 grouped areas from 3207 sections for Vector Index display | trap_surveillance |

## Loading Functions

All shapefile loading is handled by `shared/geometry_helpers.R`:

| Function | What It Loads | Key Parameters |
|----------|--------------|----------------|
| `load_shapefile(path)` | Any single `.shp` file with CRS transform to EPSG:4326 | `shapefile_path` |
| `load_background_layers(data_dir)` | `facility_boundaries.shp` + `zone_boundaries.shp` | `data_dir` (default: `shared/Q_to_R/data/`) |
| `load_harborage_layers(data_dir, facility)` | All `*_harborages.shp` for a facility | `data_dir`, `facility` |
| `load_airsite_layers(data_dir, facility)` | All `*_airsites.shp` for a facility | `data_dir`, `facility` |

## Extraction Scripts

Scripts to regenerate shapefiles from the database are in `shared/Q_to_R/`:

| Script | Purpose |
|--------|---------|
| `extract_geometries_from_db.R` | **Recommended** — Full extraction of all district-wide shapefiles |
| `extract_to_csv_then_shp.R` | Two-stage export (CSV first for inspection, then convert to .shp) |
| `create_section_boundaries.R` | Minimal extraction for section boundaries only |

See [shared/Q_to_R/GEOSPATIAL_EXTRACTION.md](../shared/Q_to_R/GEOSPATIAL_EXTRACTION.md) for full usage details including run commands and prerequisites.
